# design —— 设计与 UI 参考

> 这里放**它长什么样、参考了谁**:竞品截图、灵感、UI 稿、图标素材。
> 跟 `docs/`(想清楚)和 `app/`(做出来)并列,互不混淆。

## 子目录

| 目录 | 放什么 |
|------|--------|
| [`references/`](references/) | 竞品 / 灵感 / 同类产品截图,附一句"看中它哪点" |
| [`ui/`](ui/) | 自己的 UI 设计稿、线框图、Figma 导出 |
| [`assets/`](assets/) | 图标、Logo、配色、字体等成品素材 |

## 一点建议

- 存参考图时,**顺手写一句为什么收藏它**,否则过后只剩一堆看不懂的图。
- 设计风格 / 配色一旦定下来,可在本文件顶部记一条"设计基调",让 UI 保持一致。
- 凡涉及 UI、交互、视觉呈现、页面布局、设计稿、用户体验路径,必须提供可视化预览;不要只放文字说明、ASCII 线框或 Markdown 表格。
- 优先放 OpenDesign artifact / Figma 链接等可编辑设计产物;如果工具不可用,用本地 HTML + PNG 截图或可打开图片兜底。
- OpenDesign 未安装 / 未运行、无 active project、权限不足、连接失败或 MCP 未热加载时,先问用户是否需要帮忙安装 / 启动 / 授权 / 注册 MCP / 重载或新开会话;没有 active project 时要求用户在 OpenDesign 内创建或点进项目;用户不想处理 OpenDesign 时,直接按用户偏好走兜底预览。
- 设计说明和可视化预览都要留路径;如果用兜底方案,写清 OpenDesign 当前状态和后续恢复条件。
