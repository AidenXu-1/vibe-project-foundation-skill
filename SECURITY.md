# 安全说明

`vibe-project-foundation` 是本地脚手架 Skill。它不需要 API Key、账号密码或第三方服务凭证。

## 不要提交这些内容

- `.env` 或真实密钥。
- 用户私有项目文件。
- 生成出来的新项目目录。
- 聊天记录、客户资料、内部文档。
- 本地缓存、`.DS_Store`、`__pycache__`。

## 安全原则

- 脚本必须拒绝覆盖非空目录。
- 脚本失败时应清理本次新建的目标目录。
- 生成模板不应包含真实凭证或生产配置。
- `scratch/` 用于实验，默认不进入 Git。

## 反馈安全问题

请优先使用 GitHub Security Advisory。  
如果只能发 Issue，请不要贴出真实密钥、私有数据或可利用细节。
